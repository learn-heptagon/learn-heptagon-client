Property Verification

Using Heptagon's syntax, each node may be associated with an assume/guarantee contract. For instance, in the editor below, the two switch nodes implement a flip-flop gate: setting set to true sets the output to true, while setting rs resets the output to false. The contract of switch2 specifies this behavior, assuming that set and rs are never true at the same time, which is unspecified. Nodes switch1 and switch2 are equivalent when this unspecified behavior does not occur.

Clicking the new "Verify properties" tab checks that these assume-guarantee contracts hold, using standard model-checking techniques (more in the presentation). You should see some green text, and a green marker in the margin of the editor for each verified property. Feel-free to modify the properties or the implementation and restart the verification to see what breaks these properties.
